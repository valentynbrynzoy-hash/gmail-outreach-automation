# 📬 Outreach блогеров — Google Apps Script

Система автоматической холодной рассылки по Instagram-блогерам для партнёрского аутрича, построенная на Google Apps Script + Google Sheets.

Закрывает полный цикл: холодное письмо → фолоу-апы в треде → детектирование ответов → обработка bounces — всё по местному времени блогера с умной рандомизацией слотов, чтобы письма не кластеризовались по времени отправки.

---

## Что умеет

- **Плановая рассылка** — отправляет в местное время блогера, конвертируя из его гео-таймзоны в UTC
- **Фолоу-апы в треде** — отвечает внутри оригинального треда через Gmail API с корректными заголовками `In-Reply-To`
- **Умный рандомайзер времени** — распределяет фолоу-апы в окне ±45 мин, минимальный зазор между слотами 3 мин
- **Детектирование ответов** — трёхпроходный поиск: по треду, по теме письма, по домену отправителя
- **Обработка bounces** — парсит заголовки mailer-daemon / postmaster, помечает строки автоматически
- **Ротация шаблонов** — отслеживает использованные ID по каждому аккаунту, не повторяет в заданном окне
- **Мультиаккаунт** — один экземпляр скрипта на каждый Gmail; все аккаунты работают с одной таблицей
- **Батч-обработка** — постоянный курсор батча на аккаунт для проверки ответов (не выбивает лимиты Gmail API)
- **Авточистка лога** — еженедельный крон удаляет записи старше N дней
- **Статус-дашборд** — ячейка статуса триггера для каждого аккаунта на листе Меню

---

## Скрипты

| Файл | Назначение |
|------|-----------|
| `Code.gs` | Основной скрипт — рассылка, фолоу-апы, проверка ответов, bounces. Один экземпляр на каждый Gmail-аккаунт |
| `BatchFiller.gs` | Вспомогательный скрипт — автозаполнение даты, аккаунта и времени отправки по новым строкам блогеров. Запускается вручную через меню таблицы |

---

## Архитектура

```
Google Sheet (общая)
├── 📋 Меню          — дашборд, список аккаунтов, настройки
├── 📮 Почты         — строки блогеров (одна строка = один блогер)
├── 📧 Шаблоны       — профили отправителей + шаблоны холодных писем
├── 🔁 Фолоу-апы     — шаблоны фолоу-апов (по номеру ФА + стилю)
└── 📊 Логи          — лог отправок / проверок / bounces
```

Каждый Gmail-аккаунт запускает свою копию `Code.gs` в отдельном Apps Script проекте, привязанном к одной и той же таблице. Единственное отличие между копиями — значение `MY_ACCOUNT`.

## Установка

### 1. Создай таблицу

Создай новую **Google Таблицу** (не импортируй xlsx напрямую).

В репозитории лежит `SPREADSHEET_TEMPLATE.xlsx` — он нужен только как визуальный образец: названия листов, заголовки колонок, форматирование. Структуру воссоздавай руками в Google Sheets по [`SPREADSHEET_STRUCTURE.md`](SPREADSHEET_STRUCTURE.md).

> Импорт xlsx в Google Sheets не гарантирует корректную работу — формулы, именованные листы и форматы ячеек нужно настраивать с нуля.

### 2. Создай Apps Script проекты

**`BatchFiller.gs`** — через таблицу: **Расширения → Apps Script**. Вставь содержимое `BatchFiller.gs`. Этот скрипт привязан к таблице и добавляет меню.

**`Code.gs`** — только через [script.google.com](https://script.google.com) как отдельный standalone-проект. Важно: проект должен быть открыт под тем Gmail-аккаунтом, который указан в `MY_ACCOUNT` — только тогда скрипт будет отправлять письма от его имени. Для каждого аккаунта — свой проект.

### 3. Настрой скрипт

Отредактируй верхний блок в `Code.gs`:

```javascript
const SPREADSHEET_ID = 'ID_ТВОЕЙ_ТАБЛИЦЫ';      // из URL таблицы
const MY_ACCOUNT     = 'твой_аккаунт@gmail.com'; // Gmail, от имени которого работает эта копия

const ACCOUNT_ROW = {
  'твой_аккаунт@gmail.com':   6,   // номер строки в 📋 Меню для этого аккаунта
  'второй_аккаунт@gmail.com': 7,
  // добавляй по необходимости
};
```

`SPREADSHEET_ID` — длинная строка из URL таблицы:
```
https://docs.google.com/spreadsheets/d/SPREADSHEET_ID/edit
```

### 4. Включи нужные сервисы

В редакторе Apps Script: **Сервисы → Добавить** → включи **Gmail API** (нужен для `sendReplyInThread`).

### 5. Заполни таблицу

- **📧 Шаблоны** → строка профиля отправителя для `MY_ACCOUNT` (отметь активным `✅`)
- **📧 Шаблоны** → хотя бы один активный шаблон письма
- **🔁 Фолоу-апы** → шаблоны для ФА №1, №2, №3
- **📮 Почты** → строки блогеров: имя, гео, дата, местное время, email, аккаунт, стиль шаблона

### 6. Настрой параметры

В **📋 Меню**, строки 19–32, заполни пары ключ-значение:

| Ключ | По умолчанию | Описание |
|------|-------------|----------|
| `FOLLOWUP_ENABLED` | `TRUE` | Включить/выключить фолоу-апы |
| `FOLLOWUP_COUNT` | `3` | Максимум фолоу-апов на блогера |
| `FOLLOWUP_INTERVAL_1` | `1` | Дней после холодного письма до ФА1 |
| `FOLLOWUP_INTERVAL_2` | `5` | Дней после ФА1 до ФА2 |
| `FOLLOWUP_INTERVAL_3` | `7` | Дней после ФА2 до ФА3 |
| `CHECK_REPLIES_ENABLED` | `TRUE` | Включить проверку ответов |
| `CHECK_REPLIES_DAYS` | `30` | Сколько дней назад проверять |
| `CHECK_BATCH_SIZE` | `150` | Строк за один запуск проверки |
| `DAYS_UNIQUE` | `5` | Дней до повторного использования холодного шаблона |
| `DAYS_UNIQUE_FOLLOWUP` | `5` | Дней до повторного использования шаблона ФА |
| `BOUNCE_CHECK_ENABLED` | `TRUE` | Включить bounce-детектор |
| `LOG_RETENTION_DAYS` | `30` | Сколько дней хранить лог |
| `FOLLOWUP_STOP_ON_REPLY` | `TRUE` | Остановить цепочку ФА при получении ответа |

### 7. Создай триггеры

Запусти `createTriggers()` один раз из редактора Apps Script (Запустить → `createTriggers`). Создаются триггеры:

| Функция | Расписание |
|---------|-----------|
| `sendScheduledEmails` | Каждую минуту |
| `sendFollowUps` | Каждую минуту |
| `checkReplies` | Каждые 3 часа |
| `checkBounces` | Каждые 6 часов |
| `updateTriggerStatus` | Каждые 5 минут |
| `cleanLog` | Воскресенье в 3:00 |

## Переменные шаблонов

Используй в теме и теле письма:

| Переменная | Подставляется |
|-----------|---------------|
| `{{блогер}}` | Имя блогера (колонка A листа 📮 Почты) |
| `{{имя_отправителя}}` | Имя отправителя |
| `{{подпись}}` | Блок подписи отправителя |
| `{{ватсап}}` | Строка WhatsApp отправителя |
| `{{тема_оригинала}}` | Тема холодного письма (только для ФА) |

---

## Режимы строк

| Режим | Поведение |
|-------|-----------|
| `полный` | Отправка + фолоу-апы + проверка ответов |
| `только проверка` | Без отправки, только проверка ответов |
| *(пусто)* | Устанавливается в `полный` автоматически после первой отправки |

---

## Статусы фолоу-апов

| Статус | Значение |
|--------|----------|
| `ожидание` | ФА запланирован, ждёт времени отправки |
| `ответил` | Получен ответ, цепочка остановлена |
| `завершён` | Все ФА отправлены |
| `нет треда` | Оригинальный тред не найден в Отправленных |
| `bounce` | Письмо вернулось, остановлено |

## Служебные функции

| Функция | Что делает |
|---------|-----------|
| `createTriggers()` | Создаёт все триггеры (запускать один раз при настройке) |
| `deleteTriggers()` | Удаляет все триггеры |
| `resetStatus()` | Сбрасывает статус отправки по всем строкам |
| `cleanLog()` | Запускает очистку лога вручную |

## Поддерживаемые гео

Скрипт конвертирует местное время отправки в UTC по захардкоженным смещениям:

`ПЕРУ` (-5), `ЧИЛИ` (-3), `АРГЕНТИНА` (-3), `КОЛУМБИЯ` (-5), `МЕКСИКА` (-6), `БРАЗИЛИЯ` (-3), `УКРАИНА` (+3), `ТАИЛАНД` (+7), `МОНГОЛИЯ` (+8), `ПАКИСТАН` (+5), `ШРИ-ЛАНКА` (+5)

Чтобы добавить новые — расширь объект `GEO_OFFSETS` в начале `Code.gs`.

## Несколько аккаунтов

Для каждого Gmail-аккаунта:
1. Открой общую таблицу под этим Google-аккаунтом
2. Создай новый Apps Script проект
3. Вставь `Code.gs`, измени `MY_ACCOUNT` на email этого аккаунта
4. Запусти `createTriggers()`

Все копии читают и пишут в одну таблицу. Каждый аккаунт обрабатывает только строки, где колонка F (`С какого аккаунта`) совпадает с `MY_ACCOUNT`.

## BatchFiller — автозаполнение новых строк

`BatchFiller.gs` — вспомогательный скрипт для быстрой подготовки новых блогеров к рассылке. Добавляется в тот же Apps Script проект, что и `Code.gs`.

После добавления в таблице появляется меню **📬 Рассылка → 🚀 Заполнить новые каналы**.

**Что делает:**
- Находит строки в 📮 Почты, где есть имя и гео, но не заполнены аккаунт и дата
- Распределяет их по активным аккаунтам равномерно (до `BF_MAX_PER_ACC` на аккаунт в день)
- Проставляет дату отправки, аккаунт и местное время с шагом `BF_INTERVAL_MIN` минут начиная с `BF_START_TIME`
- Умеет пропускать выходные

**Настройки** (верхний блок `BatchFiller.gs`):

| Константа | По умолчанию | Описание |
|-----------|-------------|----------|
| `BF_INTERVAL_MIN` | `5` | Минут между письмами одного аккаунта |
| `BF_START_TIME` | `09:00` | Время старта (украинское) |
| `BF_MAX_PER_ACC` | `10` | Максимум писем на аккаунт в день |

## Важно

- Не коммить в публичный репозиторий реальный `SPREADSHEET_ID` и Gmail-адреса — используй плейсхолдеры как в `Code.gs`
- Суточные лимиты Gmail API применяются; скрипт ограничивает себя до 200 поисковых вызовов за запуск через `GMAIL_CALL_LIMIT`
